# Incident Process: Hypothesis-Driven Debugging

- Begin with clear triage: gather context, logs, and symptoms.
- Formulate initial hypotheses based on observed evidence.
- Prioritize hypotheses by likelihood and impact.
- Design validation steps for each hypothesis.
- Collect additional data to confirm or refute hypotheses.
- Attempt to reliably reproduce the issue in a controlled environment.
- Identify and implement the root cause fix.
- Validate the fix with regression and edge case tests.
- Document findings, actions taken, and lessons learned.
- Add or update regression tests to prevent recurrence.
- Review and improve monitoring and alerting as needed.
- Share knowledge with relevant teams.
